# Emails Fonctionnels — Collecte d'informations Key Users

> Modèles d'emails orientés métier à envoyer aux Key Users pour collecter les informations fonctionnelles nécessaires à la cartographie SI.
> Les emails techniques détaillés sont disponibles dans `EMAILS_COLLECTE_INFO.md`.

---

## Modèle générique

```
Objet : [Cartographie SI] Votre système [NOM DU SYSTÈME] — Collecte d'informations

Bonjour,

Dans le cadre du projet de cartographie du Système d'Information d'Air Algérie,
vous avez été désigné(e) référent(e) pour le système [NOM DU SYSTÈME].

Afin de documenter au mieux cet outil, nous aurions besoin de votre retour
sur les points suivants :

1. À quoi sert ce système au quotidien dans votre activité ?
2. Quelles sont les principales fonctionnalités que vous utilisez ?
3. Avec quels autres outils ou systèmes échangez-vous des données ?
   (import/export de fichiers, saisie manuelle, synchronisation automatique)
4. Quelles difficultés rencontrez-vous régulièrement ?
5. Si ce système tombe en panne, quel est l'impact sur votre travail ?

Nous vous serions également reconnaissants de nous fournir une extraction
de données anonymisées (exports, captures d'écran sans données personnelles,
exemples de rapports) pour mieux comprendre le fonctionnement de l'outil.

Si vous le souhaitez, nous pouvons fixer un rendez-vous pour en discuter
de vive voix. N'hésitez pas à nous proposer un créneau.

Cordialement,
Direction des Systèmes d'Information
Air Algérie
```

### 1. Objectif du projet

L'objectif est de constituer une **cartographie complète et à jour** de l'ensemble des systèmes d'information utilisés au sein d'Air Algérie, incluant :
- L'inventaire des systèmes et applications
- L'identification des référents (Key Users) par système
- La documentation des flux de données entre systèmes
- L'évaluation de la criticité de chaque système

Pour chaque système identifié, un **Key User Principal** et un **Key User Suppléant** ont été désignés par les structures concernées. Ces référents sont les interlocuteurs privilégiés pour la collecte d'informations.

Deux types de sollicitations ont été préparés :
- **Emails fonctionnels** (ce document) : questions orientées métier, accessibles, adressées directement aux Key Users avec leurs noms et coordonnées
- **Emails techniques** (document `EMAILS_COLLECTE_INFO.md`) : templates génériques détaillés par catégorie de système, destinés aux experts techniques
